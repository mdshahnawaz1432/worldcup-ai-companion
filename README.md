# WorldCup AI Companion ⚽🤖

A state-of-the-art, production-grade Android companion application built for the World Cup. Powered by the **TxLINE Real-Time API Stream** and the advanced **Gemini-3.5-Flash** intelligence layer, this app brings real-time live match statistics, dynamic betting odds, dynamic group standings, local SQL data persistence, system push notifications, and AI-driven match analysis into a single unified workspace.

---

## 🌟 Major Highlights & Core Capabilities

1. **TxLINE Live Matches & Real-Time Polling**
   - Implements continuous data retrieval modeling real-time REST / SSE (Server-Sent Events) feeds.
   - Dynamic match tick simulation that updates match minutes, injects yellow cards, and triggers goals.
2. **Match Details Dashboard**
   - **Overview Tab:** Displays venue, competition info, kickoff times, live odds, and the Solana Prediction Portal.
   - **Stats Tab:** Uses advanced container-based custom progress meters comparing team possession, total shots, expected goals (xG), corners, and cards.
   - **Events Tab:** Timeline displaying goals, substitutions, and bookings with corresponding minute markers.
   - **AI Insights Tab:** Deep analytical match breakdowns generated by Gemini.
3. **Standings & Teams Directory**
   - Dynamic calculations of team standings (Points, Matches Played, Goals For/Against, and form history) updated in real time.
   - Interactive search queries matching team names, shortcodes, and group classifications.
4. **Gemini AI Intelligence Layer**
   - Integration with Google’s `gemini-3.5-flash` using official web API structures via OkHttp and Moshi.
   - Generates contextual momentum charts, spatial tactical analysis, standout player ratings, and win probability estimates (distinctly labeled as analytical estimates).
5. **Local SQLite Room Persistence**
   - High-performance caching of saved matches and team bookmarks.
   - Direct toggles synced immediately to offline storage to prevent loss of state.
6. **System Push Notifications**
   - Subscribed to TxLINE goal broadcasts.
   - Triggers native Android system notification cards with sound/heads-up display when a goal is scored, complete with scorer names, assisting players, and scoring details.
7. **Web3 Solana Wallet integration**
   - In-app adapter simulator that integrates popular Solana mobile wallets (Phantom, Solflare, Backpack).
   - Simulates wallet connection, retrieves balances, and signs predictive transactions on-chain.
8. **Settings & Customization**
   - Smooth system-wide **Dark/Light Mode** theme transitions.
   - Custom **Polling Intervals** (choose between 15s, 30s, or 60s live stream refreshes).
   - Dynamic language locale adapters supporting English, Español, and Português.

## 🏗️ Clean Software Architecture (MVVM)

The app is meticulously structured in line with standard **Android Clean Architecture** guidelines to separate concerns:

- **`data/model/` (Domain Models):** Holds core schemas including `Match`, `MatchStats`, `MatchEvent`, `Team`, `Odds`, and SQLite `@Entity` Room annotations for local bookmarks.
- **`data/local/` & `data/repository/` (Persistence Layer):** Standard repository pattern querying Room `FavoriteDao` and caching favorites securely.
- **`data/remote/` (API & AI Service Layer):**
  - `TxLineService`: Drives the sandbox and live match streams. Implements robust Guest JWT authorization and resilient `retryIO` failover wrappers.
  - `GeminiService`: Handles secure API handshakes to Google Generative AI endpoints, generating prompt context and parsing valid structural JSON blocks with full fallback capabilities.
- **`ui/viewmodel/` (Presentation & State Engine):** Unifies data flows using standard Kotlin `MutableStateFlow` and structured Coroutines. Tracks user configurations, active game polling, and notification triggers.
- **`ui/screens/` & `ui/theme/` (UI Layer):** Powered entirely by **Jetpack Compose** following **Material 3** guidelines, utilising edge-to-edge screens (`enableEdgeToEdge`), dynamic custom styling, fluid grids, and accessible 48dp touch targets.

## 🔑 Environment Setup & Secrets Configuration

All API keys are securely managed via the **Secrets panel in AI Studio** and accessed at runtime via `BuildConfig`. Do **not** commit keys directly to the repository.

1. Navigate to the Secrets panel in AI Studio.
2. Add a new secret with the name: **`GEMINI_API_KEY`**.
3. Set your active Google AI Studio Gemini API key.
4. For local testing, copy the template from `.env.example`:
   ```bash
   cp .env.example .env
   ```
